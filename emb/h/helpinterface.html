help Open/Close info panel

If you are reading this, then you have already harnessed half of this button's power.

settings Open/Close settings panel

Change various settings here. Pick which corpora to search. Set limits on results. Choose how to display the various results you will get.

handyman Open/Close lexical tools panel

Look up words in the dictionary. Parse words.

cancel Clear session

This will reset all session variables. Strong stuff. Search criteria, etc. will all be cleared. All values will be restored to their defaults. Many of these defaults can be set in ~/hgs-prolix-conf.json.

Add to search list.

Whatever you see in the selection boxes will be added to the searching within list. See the tab Searching: Building search lists for details.

Put on the exclusion list.

Whatever you see in the selection boxes will be added to the list of things that will not be searched. See the tab Searching: Building search lists for details.

Show additional selection criteria

Make further selection criteria avalable. e.g., Search for X near Y and within N words/lines

Hide additional selection criteria

Make further selection criteria unavalable.

Perform a search

Click to execute a search based on the information in the neighboring boxes.

Browse this location

Open a text browser window to explore the work and location that is presently showing in the selection fields. Incomplete selections will send you to the first valid possibility: the first line of the first work of an author; the first line of a selected work; the first chapter of that book of that work; ...

Show/Hide local info about the works of this author

Take a peek at the database info attached to this author: genre classification of the author and the individual works; date assigned; works of the author; editions from which the works are taken.

library_books Generate a simple text of this selection (only visible if a selection has been made)

Generate a simple text of the work and location that is presently showing in the selection fields. Incomplete selections will give you to the first valid possibility: the whole of the first work of an author; the whole of the first book of a selected work; ... This is less useful for works like plays that are broken up only by lines: either the whole play will be shown or just an individual line.

subject Build an index of this selection (only visible if a selection has been made)

An index for the current selection will be built. The time required increases dramatically as works get longer. And it will take a very long time indeed to compile a index for something like Cassius Dio's Historiae Romanae. Then again, how long would it take to do this by hand? As with the browser and text maker, the index maker is sensitive to the details of your selections: it will make an index of a genre, multiple authors, a single author, a whole work, or any arbitrary subdivision of a work all the way down (somewhat comically) to an individual line.

format_list_numbered Build a vocabulary list for this selection (only visible if a selection has been made)

An vocabulary list for the current selection will be built. The time required increases dramatically as works get longer. As with the index and text maker, the vocabulary list maker is sensitive to the details of your selections: it will make an index of a genre, multiple authors, a single author, a whole work, or any arbitrary subdivision of a work all the way down (somewhat comically) to an individual line.

save Pick a save slot into which to save your current session

Five little buttons will appear (➊, ➋, ...). Click on one of them to store your current settings. Active corpora, search lists, sorting preferences, etc. will all be stored. Query terms and actual search results will not be stored. NB: your browser needs to be configured to accept cookies for this to work.

folder_open Pick a save slot from which to load stored session

Five little buttons will appear (➀, ➁, ...). Click on one of them to restore saved settings. Active corpora, search lists, sorting preferences, etc. will all be fetched. Query terms and actual search results will not be fetched.
